作品名　one stroke picuture(仮)

作品コンセプト・目的
展示を見てくれた人達が作り上げる一つの作品
お客さんが自分も関わってなにか残せるような作品を作りたい

体験の流れ
自由に描けるお絵かきキャンバス型でタブレットに絵を描く
タブレットに書かれたものを送信する
スクリーンにリアルタイムで表示されるようにする
ウェブサイトで離れていても絵を見ることができる


ウェブサイト
記録的なギャラリー
どのように絵が書き加えられてきたかのログ
とりあえずは見るだけにしようと思う
現在のキャンバスの状況



時間で区切っても面白いかも
キャンパスをなるべく大きくするべきかも

一人一人の絵を反映していく，ひとりひとつのビルが建っていく


システム考案
バックエンド：サーバ（flask?），データベース（SQlite？firebase?）
キャンバス表示:p5.js
投稿UI :HTML，CSS.JS，React
表示用のWebサイト：React
展示環境：ブラウザとスクリーン

データ仕様
HTTP　

JSON
{
  "lines": [
    {
      "points": [
        {"x": 120, "y": 200},
        {"x": 125, "y": 205},
        {"x": 130, "y": 210}
      ],　　　　　　　　　　//天の配列
      "color": "#FF8800",　　//色
      "thickness": 3,　　　　//太さ
      "timestamp": "2025-04-01T12:00:00"　//時刻
    }
  ]
}


アート的な演出
時間によってもともとの背景が変わるようにする
背景は同じ場所で時間によって朝夕方夜のように変化するとか
その背景に絵を描くようにする

平面から立体になるようにする

線の本数の限界を決める
時間じゃやなくて何本線を書いたかでキャンバスが変わる　eg.1000本で1枚の絵





分担
バックエンド：データ保存，API，ログ取得，管理機能
投稿UI：タブレット画面からの描画，プレビュー，送信UIなど　消す機能いらない　キャンバスは投稿画面から見えないようにする，感覚で書いてもらう
キャンバス表示：p5.jsで描画処理，背景演出，ログ再生など
デザイン：色，テーマ，レイアウト
閲覧ページ担当：現在のキャンバスの状況，記録的なギャラリー，どのように絵が書き加えられてきたかのログ
